# Bearing Agent Guidance

Bearing is `@kontourai/bearing`: the portfolio's evidence-backed model
capability intelligence product. It validates provenance-bearing observations,
compiles deterministic versioned catalogs, and ranks only candidates supplied
by a caller. It never configures credentials, discovers a caller's runtime, or
invokes a model.

## Source Of Truth

- Vocabulary and boundaries: `CONTEXT.md`.
- Public types and behavior: `src/`.
- Tests: `tests/`.
- Current design decisions: `docs/decisions/`.
- `dist/` is generated by `npm run build`; do not hand-edit it.

## Invariants

- No universal model-quality score.
- Every observation carries evidence, freshness, and uncertainty.
- Model revision, quantization, runtime, adapter, tool surface, context,
  hardware, task, and evaluator dimensions are never inferred from a model
  display name.
- Unknown dimensions are explicit `null`, not silently omitted.
- Catalog compilation is deterministic for identical inputs and `asOf`.
- Conflicting facts are retained and surfaced. Samples are never rewritten as
  facts.
- Ranking and catalog reads perform no network access or model invocation.
- Bearing owns model-domain normalization, not source fetching, extraction,
  review policy, trust status, scheduling, local configuration, or invocation.

## Release Discipline

- `npm run verify` must pass locally before publication.
- `.github/workflows/release-please.yml` and `publish-npm.yml` are manual-only
  until issue #12 enables automatic triggers and the npm trusted publisher.
- The Release Please workflow must not dispatch npm publication before issue
  #12; use the locally verified publication path during that period.
- Do not treat an installed workflow as executed CI evidence.
- Do not add automatic workflow triggers while hosted CI is out of budget.
